## 升级与自动备份

1. 在 `0.4.1` 中准备任务、规则、提示词、审稿、AI 产物、Cookie 和模型 Key。
2. 关闭应用并覆盖安装 `小红书运营台-0.5.0-Setup.exe`。
3. 确认原数据、加密凭证、扩展配对和账号状态未丢失。
4. 打开“模型设置 → 备份与迁移”，确认当天自动备份存在。
5. 关闭并再次启动应用，确认同一天没有重复创建日备份。
6. 点击“立即备份”，确认出现独立的手动备份。

## 本机恢复

1. 记录当前任务、规则和产物数量，然后创建手动备份。
2. 新增一条容易识别的测试数据。
3. 对旧备份点击“预览恢复”，核对类型、时间、Schema 和警告。
4. 确认恢复；如有运行任务，验证应用先提示并安全暂停。
5. 应用自动重启后，确认新增测试数据已回到备份时状态。
6. 确认 Cookie、模型 Key 和扩展配对在相同 Windows 用户下继续可用。
7. 确认恢复前安全备份已生成，可用于反向恢复。

## 脱敏迁移

1. 点击“导出迁移包”，确认显示文件未加密和可信渠道提示。
2. 将 `.xhsmigrate` 文件导入另一个 Windows 用户或干净测试机。
3. 预检后确认业务数量正确，再执行完整替换。
4. 重启后检查任务、笔记、规则、提示词、审稿和 AI 产物。
5. 确认小红书账号、Cookie、模型 Key和扩展配对均需要重新配置。
6. 重新连接小红书并填写模型 Key，完成一次周十五蜂蜜露 AI 审稿。

## 失败保护

1. 修改迁移包任意字节，确认系统拒绝导入且当前数据库不变。
2. 使用错误扩展名或非运营台文件，确认系统给出可理解错误。
3. 使用高于当前支持 Schema 的测试包，确认系统拒绝覆盖。
4. 恢复过程中取消任务暂停提示，确认应用和当前数据库保持不变。
5. 验证最近仅保留 7 个日备份和 3 个安全备份，手动备份不会自动删除。
