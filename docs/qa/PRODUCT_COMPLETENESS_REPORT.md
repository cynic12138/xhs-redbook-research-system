## 已修复问题

### QA-001 · P1 · 目标任务无法从暂停抓取恢复

- 复现：目标任务已有暂停/失败抓取任务时点击“从失败处重试”。
- 原因：重试继续等待原任务，但没有恢复 Job worker。
- 修复：仅在复用已有暂停/失败任务时调用恢复逻辑，再继续等待入库。
- 回归：新增暂停任务恢复测试，验证沿用原 `jobId` 并完成后续产物。

### QA-002 · P1 · 后补资料不会进入研究档案

- 复现：目标任务完成后添加新的官网或媒体链接。
- 原因：公开资料及下游步骤已标记完成，重试会直接跳过。
- 修复：补充来源后失效公开资料、档案、写稿、审稿和归档步骤；保留小红书证据和历史产物，重新确认后生成新结果。运行中禁止并发补充来源。
- 回归：新增完成后补充资料及运行中拒绝修改测试。

### QA-003 · P2 · 笔记库移动端工具栏与筛选区重叠

- 复现：390px 宽度打开笔记库。
- 原因：笔记面板隐式网格列被子元素最小宽度撑大，标题区又只保留单行高度。
- 修复：面板列使用 `minmax(0, 1fr)`；工具按钮宽屏四列、手机两列；手机标题区保留两行按钮高度。
- 回归：面板 `clientWidth` 与 `scrollWidth` 一致，按钮不再覆盖搜索框。

### QA-004 · P2 · AI 助手手机端标题断成竖排

- 复现：390px 宽度打开 AI 助手。
- 修复：助手标题固定为不可折行的独立区域，模型和关闭操作继续自适应换行。
- 回归：助手抽屉宽度 330px，内容无横向溢出，标题完整显示。

### QA-005 · P2 · 未知 API 返回非结构化内容

- 复现：请求不存在的 `/api/*` 地址。
- 修复：API Router 末尾增加统一 JSON 404。
- 回归：新增未知 API 合同测试。

### QA-006 · P1 · 生产启动脚本指向不存在的构建文件

- 复现：构建后运行 `npm start`。
- 原因：脚本指向 `dist/server/index.js`，实际 TypeScript 输出为 `dist/server/server/index.js`。
- 修复：启动脚本改为实际构建入口，并新增由 `tsconfig.server.json` 推导入口路径的合同测试。
- 回归：在空临时数据目录和独立端口启动生产构建，同时验证 API 健康检查和前端静态入口。
