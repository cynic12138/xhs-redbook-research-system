# 浏览器验收清单

## 每次发布前

1. 运行 `npm run typecheck`、`npm test`、`npm run build` 和 `git diff --check`。
2. 分别使用 1440x900、1024x768、768x1024、390x844 打开本地前端。
3. 逐一进入总览、话题研究、笔记库、爆款拆解、受众洞察、竞品分析、评论运营、内容创作台、提示词和 AI 工作台。
4. 检查页面根节点无横向滚动，标题、按钮、筛选项、表格、抽屉和弹窗不互相覆盖。
5. 检查空状态、加载状态、失败状态、禁用状态和重复点击保护。

## 关键闭环

- 登录：检测扩展、同步 Cookie、验证失效提示和重新连接。
- 抓取：创建最小任务、查看进度、暂停、恢复、限流提示和数据集范围。
- 笔记：分页、筛选、详情、媒体刷新、单选、多选、批量审稿和安全删除预览。
- 分析：单篇深拆、多篇对比、受众报告、竞品报告和评论需求。
- 内容：规则库 CRUD、版本恢复、项目素材、批量写稿、批量审稿、终稿归档。
- 提示词：系统提示词、我的提示词、普通配置、高级模板、变量校验和版本恢复。
- AI：普通问答、关键词编排、目标任务确认、失败重试、补充来源、证据档案和工作台产物。

## 禁止事项

- 不使用真实账号执行发布、评论、点赞、收藏或关注操作。
- 不在截图、日志、报告和提交中保留 Cookie、Key 或其他敏感值。
- 不用高并发、高页数反复请求小红书；真实联调固定使用最小只读参数。
- 不删除非本轮创建的 QA 数据。
