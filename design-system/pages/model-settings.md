# 模型设置设计规则

> 这是全局设置抽屉规则，不是一级导航页面。适用于顶部“模型设置”入口。

## 1. 目标

模型设置用于管理 OpenAI-compatible 模型配置，不应干扰 AI 工作台阅读产物。

## 2. 推荐结构

- 右侧抽屉，宽度 560-640px。
- 顶部：标题、默认模型摘要、关闭按钮。
- 主体上半：已配置模型列表。
- 主体下半：新增/编辑表单，默认收起。
- 底部：测试连接状态与安全说明。

## 3. 已配置模型列表

- 每条显示：名称、provider、model、masked key、默认状态、测试、编辑、设默认、删除。
- 默认模型用绿色或蓝色 chip 标识。
- 测试中显示 loading；测试结果显示成功/失败原因。
- 删除按钮使用危险样式并远离主操作。

## 4. 新增/编辑表单

- 字段：名称、Provider、Base URL、Model、API Key。
- 每个字段必须有 label，不能只用 placeholder。
- API Key 输入框默认 password，不回显明文。
- 保存成功后收起表单并刷新列表。

## 5. 安全规则

- 前端永不展示完整 API Key。
- 不把 API Key 写入 Prompt 或 AI 产物。
- 测试连接失败时只展示必要错误，不打印密钥。

## 6. 禁用设计

- 禁止出现横向滚动条。
- 禁止把模型设置放回 AI 工作台主阅读区。
- 禁止一个大表单占满整个弹窗。
