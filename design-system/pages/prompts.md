# Prompt 中心设计规则

> 页面 key：`prompts`。用于查看、编辑、启用、恢复和追踪 AI 工作流 Prompt。

## 1. 页面目标

Prompt 中心要让用户明确知道：

- 每个工作流使用哪个 Prompt。
- 当前启用的是默认版本还是自定义版本。
- Prompt 需要哪些变量。
- 输出结构是什么。
- 这个 Prompt 生成过哪些 AI 产物。

## 2. 推荐布局

- 左侧：Prompt 列表，宽度 240-280px。
- 中间：Prompt 编辑工作区，占主要宽度。
- 右侧：关联 AI 产物，宽度 260-320px。
- 中间编辑区再分上下：
  - 顶部控制区：标题、版本、启用状态、保存、启用、恢复默认、变量、输出结构。
  - 下方大编辑器：完整 Prompt 文本。
- 每个区域都要内部滚动，不允许内容把整页撑开。

## 3. 左侧 Prompt 列表

- 显示：工作流名称、版本、默认/自定义、产物数、最后使用时间。
- 当前选中项使用粉色浅底和边框。
- 列表超过高度后内部滚动。

## 4. 顶部控制区

- 标题左侧显示工作流名称。
- 状态 chips：当前启用、默认 Prompt、自定义 Prompt、版本、产物数。
- 按钮区在右上角：默认启用/自定义启用、恢复默认、保存。
- 可用变量使用 chip 展示；输出结构使用短标签或折叠摘要。
- 该区域高度建议 140-220px，超过时内部滚动。

## 5. Prompt 编辑器

- 编辑器占据中间区主要高度。
- 使用等宽字体，行高 1.6。
- 保存前不改变启用版本，除非用户明确点击“启用自定义”。
- 恢复默认是危险操作，使用红色/警示样式。

## 6. 关联 AI 产物

- 右侧列出该 Prompt 生成的产物。
- 每条显示：产物标题、生成时间、模型、来源任务。
- 点击产物跳转 AI 工作台并打开阅读器。
- 无产物时显示“这个 Prompt 暂无关联产物”以及“去对应页面生成”。

## 7. 禁用设计

- 禁止把 Prompt 版本作为静态文字堆在 AI 工作台角落。
- 禁止用户无法判断当前到底用了哪个 Prompt。
- 禁止保存后不反馈成功/失败。
